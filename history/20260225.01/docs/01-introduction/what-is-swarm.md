# 什么是蜂群式Agent系统？

## 用生活中的例子理解

想象一下你去一家大医院看病：

```
你（患者） → 前台导诊（协调员） → 根据病情分配 → 
                              ↓
                    ┌─────────┼─────────┐
                    ↓         ↓         ↓
                  内科医生   外科医生   牙科医生  （专家Agent）
                    ↓         ↓         ↓
                  给你看病   给你看病   给你看病
```

**蜂群式Agent系统**就像这个医院：

- **协调员** = 前台导诊（先判断你是什么问题）
- **专家Agent** = 各个科室的医生（专门处理某一类问题）
- **用户** = 患者（提出问题）
- **工具库** = 医院的检查设备和病历库
- **大模型** = 超级聪明的医生顾问

## 核心特点

1. **智能协调** - 使用大模型理解用户需求，智能分配任务
2. **多专家协作** - 支持多个专业领域的Agent同时工作
3. **并发执行** - 多个专家同时工作，提高效率
4. **报告生成** - 自动生成完整的分析报告

## 与传统单Agent的区别

| 传统单Agent | 蜂群式Agent系统 |
|------------|----------------|
| 一个Agent处理所有问题 | 多个专家Agent协作 |
| 简单匹配 | 大模型智能分析分配 |
| 串行执行 | 并行执行 |
| 单一视角 | 多维度分析 |
