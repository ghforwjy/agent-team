# 认知偏差预防机制

> 如何通过基因设计防止 AI 的认知偏差

---

## 核心概念

**认知偏差**是指思维中的系统性错误，导致判断偏离理性。对于 AI 助手来说，常见的认知偏差包括：

- **确认偏误** - 只寻找支持自己观点的证据
- **可得性偏误** - 依赖最容易获取的信息
- **锚定效应** - 过度依赖第一印象
- **术语混淆** - 将不同概念混为一谈

## 问题背景

### 真实案例：Evolver 基因认知偏差

**场景回顾**：
```
用户: "你现在拥有哪些evolver基因"
AI:  [只读取了 evolver-agent.json]
     "我有4个MCP工具..."
用户: "只有一个基因吗？"
AI:  [深入探索后发现]
     "原来有4个GEP基因！"
```

**认知偏差分析**：

| 偏差类型 | 表现 | 原因 |
|----------|------|------|
| **确认偏误** | 看到 `evolution.enabled` 就认为找到了基因 | 急于确认自己的理解 |
| **可得性偏误** | 只读取最容易看到的配置文件 | 没有深入探索目录结构 |
| **锚定效应** | 被第一个文件的内容锚定 | 没有质疑初始信息源 |
| **术语混淆** | 将 "MCP工具" 和 "基因" 混为一谈 | 没有验证术语定义 |

## 解决方案：`gene_prevent_cognitive_bias`

### 基因设计

新基因 `gene_prevent_cognitive_bias` 专门用于预防认知偏差：

**触发信号**：
- `initial_assumption_made` - 做出初始假设
- `surface_answer_given` - 给出表面答案
- `user_followup_question` - 用户追问
- `incomplete_exploration` - 探索不完整
- `cognitive_dissonance_detected` - 检测到认知失调

**核心策略**（7步法）：

```
1. 识别初始假设：明确说出自己的初始理解
2. 质疑假设：列出可能错误的3个原因
3. 深度探索：检查至少3个不同位置的信息源
4. 交叉验证：对比多个来源的信息一致性
5. 承认不确定性：如果信息不完整，明确说明
6. 迭代修正：根据新信息更新理解
7. 记录偏差模式：将认知偏差类型记录到学习手册
```

### 4种认知偏差检查清单

#### 1. 确认偏误检查

**自检问题**：
- ❓ 我是否只看了支持初始假设的信息？
- ❓ 我是否主动寻找了反面证据？
- ❓ 我是否考虑了其他可能的解释？

**预防措施**：
```
强制查看相反观点的信息源
→ 如果认为"A是正确的"，先找"A可能是错误的"证据
```

#### 2. 可得性偏误检查

**自检问题**：
- ❓ 我是否因为某个信息最容易获取就使用了它？
- ❓ 我是否检查了隐藏/深层的信息源？
- ❓ 我是否探索了目录结构的各个层级？

**预防措施**：
```
强制探索不明显的信息源
→ 不要只读根目录的配置文件
→ 深入探索子目录、代码实现、文档
```

#### 3. 锚定效应检查

**自检问题**：
- ❓ 我是否被第一个看到的信息过度影响了？
- ❓ 我是否在查看任何信息前先列出了多个可能的信息源？
- ❓ 我是否定期重新审视初始假设？

**预防措施**：
```
在查看任何信息前，先列出多个可能的信息源
→ 制定探索计划，而不是随机查看
```

#### 4. 术语混淆检查

**自检问题**：
- ❓ 我是否确认了每个术语的准确定义？
- ❓ 我是否将不同概念混为一谈了？
- ❓ 我是否验证了术语在不同上下文中的含义？

**预防措施**：
```
在使用术语前，先验证其定义
→ "基因"在MCP层和GEP层是不同的概念
→ 明确区分"工具"、"接口"、"实现"
```

## 实际应用示例

### 改进后的回答流程

**用户问题**："你现在拥有哪些evolver基因？"

**旧流程（有认知偏差）**：
```
1. 读取 evolver-agent.json
2. 看到 evolution.enabled: true
3. 误以为这就是基因配置
4. 回答："我有进化能力..."
```

**新流程（应用认知偏差预防基因）**：
```
1. 识别初始假设：
   "我的初始理解是：基因可能在配置文件中定义"

2. 质疑假设（3个可能错误）：
   - 错误1: 配置文件可能只包含接口配置，不包含基因定义
   - 错误2: 基因可能在其他目录中定义
   - 错误3: 我可能混淆了"工具"和"基因"的概念

3. 深度探索（3+信息源）：
   - 源1: evolver-agent.json (配置文件)
   - 源2: evolver/assets/gep/ (基因资产目录)
   - 源3: evolver/src/gep/ (基因实现代码)
   - 源4: 目录结构分析

4. 交叉验证：
   - 配置文件只有MCP服务器配置
   - genes.json 包含基因定义
   - 实现代码验证了基因的工作方式

5. 承认不确定性：
   "在我深入探索之前，我不确定基因的确切数量和位置"

6. 迭代修正：
   "修正：MCP层没有基因，基因在GEP层的assets/gep/中"

7. 记录偏差：
   将"确认偏误"、"可得性偏误"记录到学习手册
```
